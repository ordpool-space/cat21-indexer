About CAT-21

What is a CAT-21?

CAT-21 is the cryptopunks of nLockTime. Every Bitcoin transaction whose nLockTime is exactly 21 is a CAT-21 mint. The cat's image is generated deterministically from SHA256(txid + blockId) — no off-chain server, no API key, no centralized renderer. Pure consensus data.

Each cat lives on a single Bitcoin sat. The cat number is the order in which it was mined. Cat #0 is the Genesis Cat, minted in block 824,205.

Why nLockTime?

Bitcoin Knots (Luke-jr's fork) added a policy rule to reject "parasite-cat21" transactions, officially recognising CAT-21 as worth blocking. That's mempool-relay policy, not consensus — miners still include the transactions. And if anyone blocks nLockTime=21, we move to 2121. You can't play whack-a-mole with a core transaction field.

nLockTime is a legitimate consensus field, not "data storage". BIP-110 would kill inscriptions, atomicals, and labitbu — CAT-21 survives.

Origin

Inspired by 1440000bytes' "inscriptionless inscriptions" post on Delving Bitcoin. CAT-21 took that idea and ran with it: minimal data, deterministic art, ordinals-compatible ownership tracking.

Wallet safety

Cats live on Bitcoin sats — your wallet has to know not to spend them. Confirmed-safe wallets: Xverse and Leather. Unisat is not safe (will spend cat sats). Other wallets supported on this site for read-only viewing.
